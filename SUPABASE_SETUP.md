# Supabase Database Setup Guide for Prime Pages

Follow these simple steps to set up a free Supabase database and connect it to your Prime Pages landing page and admin panel.

## 1. Create a Project on Supabase
1. Go to [https://supabase.com](https://supabase.com) and sign up for a free account.
2. Click **New Project** and name it `Prime Pages`.
3. Set a Database Password (save it somewhere) and select a region (e.g. Singapore or nearest).
4. Wait a few minutes for the project to provision.

---

## 2. Get API Keys & Paste into `admin.js` & `app.js`
1. Inside your Supabase project dashboard, click on **Project Settings** (gear icon on bottom left) -> **API**.
2. Copy your **Project URL** (under Host/URL).
3. Copy your **Project API anon key** (under `anon` / `public`).
4. Open the following files in your workspace:
   - [admin.js](file:///C:/Users/Hotspot/.gemini/antigravity/scratch/prime-pages/admin.js)
   - [app.js](file:///C:/Users/Hotspot/.gemini/antigravity/scratch/prime-pages/app.js)
5. Paste the keys at the very top under:
   ```javascript
   const SUPABASE_URL = "YOUR_PROJECT_URL_HERE";
   const SUPABASE_ANON_KEY = "YOUR_ANON_KEY_HERE";
   ```

---

## 3. Create Database Tables
Click on the **SQL Editor** tab (sql icon on the left sidebar) in your Supabase dashboard, click **New query**, paste the following SQL code, and click **Run**:

```sql
-- 1. Create Prices Table
CREATE TABLE prices (
    id integer PRIMARY KEY DEFAULT 1,
    notebook numeric DEFAULT 450,
    magazine numeric DEFAULT 850,
    travel numeric DEFAULT 950,
    album numeric DEFAULT 1800,
    softcover numeric DEFAULT 0,
    hardcover numeric DEFAULT 350,
    leather numeric DEFAULT 800,
    "paperIvory" numeric DEFAULT 4,
    "paperMatte" numeric DEFAULT 6,
    "paperLustre" numeric DEFAULT 8
);

-- Seed initial prices row
INSERT INTO prices (id, notebook, magazine, travel, album, softcover, hardcover, leather, "paperIvory", "paperMatte", "paperLustre")
VALUES (1, 450, 850, 950, 1800, 0, 350, 800, 4, 6, 8)
ON CONFLICT (id) DO NOTHING;

-- 2. Create Showcase Table
CREATE TABLE showcase (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    title text NOT NULL,
    category text NOT NULL,
    description text,
    "imageUrl" text NOT NULL,
    created_at timestamp with time zone DEFAULT timezone('utc'::text, now()) NOT NULL
);

-- 3. Create Inquiries Table
CREATE TABLE inquiries (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name text NOT NULL,
    email text NOT NULL,
    phone text NOT NULL,
    address text NOT NULL,
    specifications text NOT NULL,
    message text,
    "fileUrl" text,
    created_at timestamp with time zone DEFAULT timezone('utc'::text, now()) NOT NULL
);
```

---

## 4. Disable Row Level Security (RLS) for Public Forms
Since your customers will be submitting order inquiries and reading pricing without logging in, we need to allow anonymous access.
In the Supabase SQL Editor, run the following commands to allow anonymous access to your tables:

```sql
-- Disable Row Level Security (Allows anyone to read prices/showcase and write inquiries)
ALTER TABLE prices DISABLE ROW LEVEL SECURITY;
ALTER TABLE showcase DISABLE ROW LEVEL SECURITY;
ALTER TABLE inquiries DISABLE ROW LEVEL SECURITY;
```

*Note: For a professional production site, you can set custom policies to allow public reads and restrict writes to admin only. For your initial launch, disabling RLS is the simplest way to get up and running.*

---

## 5. Setup Storage Bucket for Original Photos
Since customers will upload high-res photos or ZIP archives, we store them in a Supabase Storage Bucket.
1. Go to the **Storage** tab in your Supabase Dashboard.
2. Click **New Bucket** and name it **`order-files`**.
3. Toggle the **Public** option to **Enabled** (so the admin can access the download links).
4. Run the following SQL policies in the **SQL Editor** to allow clients to upload files:

```sql
-- Allow anonymous uploads to your 'order-files' bucket
CREATE POLICY "Allow public upload" 
ON storage.objects 
FOR INSERT 
WITH CHECK (bucket_id = 'order-files');

-- Allow anonymous downloads from the bucket (so admin can download)
CREATE POLICY "Allow public download" 
ON storage.objects 
FOR SELECT 
USING (bucket_id = 'order-files');
```

---

## 6. Verify Setup
1. Refresh the Admin Page (`http://localhost:3000/admin.html`).
2. The warning pill at the top right should change from **Local Mock Mode** to **Supabase Live**!
3. Now, when customers submit the Order Form, the details will instantly appear under the **Inquiries** tab in your Admin Panel and in your Supabase Dashboard!
4. Clicking the **Download** button in the inquiries table will download the original high-resolution file.
